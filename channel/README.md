# channel/ — TeamPlus → Telegram bridge

Single Bun process that runs two async tasks in parallel:

- **teamplus.ts** — long-lived WebSocket to the TeamPlus instance (`TEAMPLUS_BASE`). Every
  inbound `IM_CHAT:NEW_MESSAGE` (skipping your own user id) is appended
  to the cursor and forwarded as a Telegram message.
- **telegram.ts** — long-poll `getUpdates`. Handles `/unread`, `/chats`,
  `/ping`, `/help`. Bot command menu is installed on boot.

State persists to `../state/cursors.json` per-chat:

```jsonc
{
  "myId": 1049,
  "chats": {
    "<chatId>": {
      "name": "邱子玲",          // null until first message carries ChatName
      "lastSeenMsgId": "abc123",
      "lastSeenTs": 1714319520,
      "unread": [ { "msgId": "...", "senderId": 2002,
                    "senderName": "...", "ts": 1714319600,
                    "msgType": 1, "content": "hi" } ],
      "spilled": 0                // older messages dropped past 200-cap
    }
  }
}
```

## Files

| File         | Role                                                        |
|--------------|-------------------------------------------------------------|
| server.ts    | Entry. Loads config, starts both loops, sends hello + menu. |
| teamplus.ts  | WS auth + listen + reconnect.                               |
| telegram.ts  | Long-poll + command router + sendMessage helper.            |
| cursor.ts    | Atomic load/save of `state/cursors.json`.                   |
| format.ts    | Telegram-safe chunker + live-line / unread-block formatters.|
| config.ts    | Loaders for cookies, `.telegram.json`, `.config.json`.      |

## Run

```sh
# from repo root, NOT inside channel/
../run.sh
```

The launcher ensures `bun install` has been run, then needs three things
in the project root:

| File              | Source                                                       |
|-------------------|--------------------------------------------------------------|
| `.env`            | Created interactively by `./scripts/setup_creds.sh`. Stores `TEAMPLUS_ACCOUNT` and `TEAMPLUS_PASSWORD` (chmod 600). |
| `.telegram.json`  | You create by hand: `{ "token": "...", "chat_id": "..." }`.  |
| `cookies.json`    | Auto-generated by `./scripts/refresh.sh` (which `run.sh` calls when missing). Uses [patchright](https://github.com/Kaliiiiiiiiii-Vinyzu/patchright) to drive Chromium through the login flow with the credentials in `.env`. Captcha is solved by `tesseract`. |
| `.config.json`    | Auto-written by `refresh_cookies.py`: extracts `my_id` from the TSSID JWT after a successful login. |

So a from-scratch setup is:

```sh
brew install bun tesseract              # one-time
./scripts/setup_creds.sh                # prompts for account + password → .env
# create .telegram.json with bot token + your chat_id
./run.sh                                # auto-runs refresh.sh, starts daemon
```

After cookies expire (TeamPlus rotates `TSSID` every ~7 days) just run
`./scripts/refresh.sh` again — `.env` already has the creds.
